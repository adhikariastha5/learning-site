💡 Why This Matters
  • Dot product = cosine similarity (same thing we used in autocomplete!)
  • Matrix multiplication = every neural network layer is just y = Wx + b
  • Inverse matrix A⁻¹ = solving systems of equations = linear regression in one step
  • When A is invertible: x = A⁻¹b gives the exact solution

If dapg > 0, predict home team wins. Simple! But when dapg is close to 0, this single feature isn't reliable enough. Solution: use more features — rebounds, turnovers, free throws. This is why feature engineering matters.

Chapter 2: Probability — Dealing with Uncertainty

Real data is noisy. Probability gives us the math to make decisions despite uncertainty.

Two Ways to Think About Probability

ApproachHow It WorksStrengthsWeaknesses
Frequentist Count occurrences / total trials. Test 100 bulbs → 3 defective → P(defective) = 0.03 Intuitive, easy to understand Only works for repeatable events. Need lots of data.
Bayesian Start with a belief (prior), update it with evidence. Your belief evolves. Works with limited data. Can encode domain knowledge. Prior choice is subjective.
📝 Skull Classification Example

An anthropologist classifies fossil skulls as male/female using circumference measurements. But skulls vary due to hereditary, nutritional, and environmental factors. We model this variability using probability distributions — the circumference follows a pattern, but with randomness.

P(39.5 < x < 40.5) = 0.02 → 2% chance the circumference is between 39.5 and 40.5 cm.

Chapter 3: Bayes' Theorem — The Heart of Classification

Given some evidence, what's the probability of each class? This is THE fundamental question.

P(C|x) = P(C) × P(x|C) / P(x)
  Bayes' Theorem — Decoded

  P(C|x)  = What we want     → "Given this data, which class?"
  P(C)    = Prior probability → "How common is this class in general?"
  P(x|C)  = Likelihood        → "If it IS this class, how likely is this data?"
  P(x)    = Evidence          → "How likely is this data overall?"
📝 Classic Example: Cold Detection

Question: A person has a fever. What's the probability they have a cold?

P(cold) = 0.01          → 1% of people have a cold
P(fever|cold) = 0.4     → 40% of cold people have fever
P(fever) = 0.02         → 2% of all people have fever

P(cold|fever) = (0.01 × 0.4) / 0.02 = 0.20 = 20%

Before knowing about the fever, there was only a 1% chance of a cold. After observing the fever, it jumped to 20%. That's the power of Bayes — evidence updates beliefs.

Decision Boundaries

Where do you draw the line between "classify as A" vs "classify as B"? At the point where P(A|x) = P(B|x). Everything to the left of that boundary is class A; everything to the right is class B.

💡 Unequal Costs of Error

Not all mistakes are equal. Missing cancer (false negative) is far worse than a false alarm (false positive). Bayesian decision making lets you assign different costs to different types of errors and shift the decision boundary accordingly.

K-Nearest Neighbors (KNN)

💡 The K choice trade-off

Small K (K=1): Very sensitive to noise. One mislabeled neighbor = wrong answer.

Large K (K=100): Over-smooth. You're averaging too many distant points, losing local patterns.

This is the bias-variance tradeoff — one of the most important concepts in all of ML.

Chapter 5: Clustering — Finding Structure Without Labels

What if nobody told you the classes? Clustering discovers them from the data itself.

Agglomerative (bottom-up): Start with each point as its own cluster, merge the two most similar. Repeat.

Divisive (top-down): Start with one big cluster, split. Repeat.

📝 Real Application

In 1971, Paykel used cluster analysis to group 165 depressed patients into 4 clusters: "anxious", "hostile", "retarded psychotic", "young depressive". No one labeled these — the algorithm found the groupings from symptom data alone.

Chapter 6: Artificial Neural Networks — Learning from Data

Inspired by biological neurons, but the math is surprisingly simple at its core.

The Single Neuron (McCulloch-Pitts Model, 1943)

  A single neuron: weighted sum → threshold → output

   x₁ ──w₁──┐
             │
   x₂ ──w₂──┼──→ [ Σ wᵢxᵢ ] ──→ [ > T? ] ──→ output (0 or 1)
             │
   x₃ ──w₃──┘

   With bias (modern form):

   output = 1  if  w₀ + w₁x₁ + w₂x₂ + ... + wₘxₘ > 0
   output = 0  otherwise

   w₀ = -T (threshold becomes a learnable bias weight)

That's it. Every deep learning model, no matter how complex, is built from this: multiply inputs by weights, sum them, apply a threshold.

Multi-Layer Networks & Backpropagation

  A neural network with hidden layers

  Input Layer      Hidden Layer     Output Layer
  (features)       (learned)        (prediction)

    x₁ ──────┐  ┌── h₁ ──┐
              ├──┤        ├──── y₁  (class A?)
    x₂ ──────┤  ├── h₂ ──┤
              ├──┤        ├──── y₂  (class B?)
    x₃ ──────┘  └── h₃ ──┘

  Backpropagation: How it learns

  Forward:  Input → Hidden → Output → Compare with truth → Error
  Backward: Error → Adjust output weights → Adjust hidden weights
            (chain rule of calculus propagates error backward)
💡 Why hidden layers matter

A single neuron can only learn linear boundaries (straight lines). With a hidden layer, the network can learn any continuous function — curves, circles, complex decision boundaries. This is the Universal Approximation Theorem, and it's why deep learning works.

Key Properties of Biological vs Artificial Neural Networks

Biological BrainArtificial Neural Network
Parallel computation (millions of neurons at once)GPU parallel processing mimics this
Specialized regions for different tasksDifferent layers learn different features
Layered architecture, mostly one-directionFeedforward: input → hidden → output
Robust to damage (graceful degradation)Dropout training creates similar robustness
